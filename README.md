# Musik Am Guertel

Minimal black-and-white editorial register for upcoming concerts around Vienna's Guertel.

## Product rules

- Future events only (`date >= today` in `Europe/Vienna` timezone)
- Chronological feed grouped by day
- No filters, no search, no accounts, no recommendations
- Event fields: `location`, `title`, `description`, `date`, `time`, `event_url`
- UI microcopy tokens: `TODAY`, `TOMORROW`, `NO UPCOMING EVENTS`, `SOURCE`

## Stack

- Vite + React + TypeScript
- Tailwind CSS
- Static build output (`dist/`)
- Data generated by scraper pipeline into `src/data/events.json`

## Local development

```bash
npm ci
npm run dev
```

## Build pipeline

Build data + frontend bundle:

```bash
npm run build
```

Run only data generation:

```bash
npm run build:data
```

Debug Chelsea scraper output:

```bash
npm run scrape:chelsea
```

## Data flow

1. `scripts/scrapers/chelsea.ts` scrapes Chelsea events.
2. Pipeline normalizes, deduplicates, filters past events, sorts chronologically.
3. Runtime validation (`zod`) verifies output schema.
4. JSON is written to `src/data/events.json` and consumed by the homepage.

Build fails on hard scraper failures (fetch/parsing no events) to avoid deploying invalid data.

## Deployment (GitHub Pages + Actions)

Workflow file: `.github/workflows/deploy.yml`

Triggers:

- Push to `main`
- Manual (`workflow_dispatch`)
- Daily cron at `03:00 UTC`

Steps:

1. `npm ci`
2. `npm run build`
3. Upload `dist/`
4. Deploy to GitHub Pages

## Custom domain

Set repository variable `CUSTOM_DOMAIN` to your domain (for example, `events.example.com`).
The workflow writes `dist/CNAME` automatically when the variable is set.

DNS setup:

- Subdomain: create a `CNAME` record to `<username>.github.io`
- Apex/root domain: configure `A`/`ALIAS` records as documented by GitHub Pages
